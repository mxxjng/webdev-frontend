E The Document Object Model

E1: Elemente finden und ändern

Ändern Sie die im HTML dargestellten Elemente ausschließlich mit Javascript (keine Attribute, Klassen oder Ids im HTML hinzufügen..). Styles auch nur per Javascript!

  • 1. Selektieren Sie die Elemente über ihre Ids, Klassen oder Tags.
  • 2. Ändern Sie die Überschrift.
  • 3. Füllen Sie das input-Element mit dem Namen js-fill per Javascript aus.

    Mit inputElement.value = newValue können sie den Text des input-Elements ändern.

  • 4. Ändern Sie die Größe des Bildes mit der Klasse gary, indem sie das Style-Attribut entfernen.
  • 5. Geben Sie beiden Bildern einen roten Rahmen, indem Sie ihnen die Klasse red-border geben
  • 6. Fügen Sie dem Bild mit der id tribore ein Alt-Attribut hinzu.
  • 7. Geben Sie der Schrift Gary einen blauen Rahmen, ohne den Rahmen des Bildes mit der selben Klasse zu ändern und ohne auch der Schrift Tribore einen Rahmen zu geben

Man kann Attributwerte immer nur für ein Element gleichzeitig ändern. die Zugriffsfunktionen mit "getElements.." geben aber immer ein Array (HTMLCollection oder NodeList) mit Elementen zurück. Man kann dann mit [0] auf das erste Element zugreifen, mit [1] auf das zweite... und auf dieses die jeweilige Methode ausführen. Wenn man mehrere Elemente aus dem Array gleich verändern möchte (5.), ist ein Loop nötig.

E3: Inhaltsverzeichnis erstellen

  • Die Vorlage enthält eine gecrawlte Wikipedia-Seite. Leider ist das Inhaltsverzeichnis abhanden gekommen und nur noch ein leeres <ul class=toc>-Element übrig.

    Erstellen sie via JavaScript automatisch das Inhaltsverzeichnis. Dazu müssen für jede h2-Überschrift folgende HTML-Elemente in das <ul class=toc> eingefügt werden: <li><a href="#id-of-h2">text-of-h2</a></li>.

    Fügen sie ihren JavaScript-Code ganz am Ende der Vorlage ein.

  • Beim crawlen sind auch Links auf andere Wikipedia-Seiten nicht richtig verarbeitet worden.

    Sie wurden nur in der Form <a href="/wiki/Cross-platform">…</a> gespeichert, wobei das href-Attribut mit https://en.wikipedia.org/wiki/ anfangen sollte. Dadurch sind die Links kaputt.

    Um weitere Prokrastination zu ermöglichen muss das behoben werden. Dazu müssen alle Links, deren href-Attribut mit /wiki/ anfängt auf https://en.wikipedia.org/wiki/ geändert werden. Verwenden sie dazu ebenfalls die DOM API.

    Verwenden sie link.getAttribute('href') statt link.href. Beim letzteren fügt sonst der Browser automatisch file:// am Begin des Strings ein.

    Der CSS-Selektor a[href^="/wiki/"] liefert ihnen alle passenden Links. Alternativ können sie sich die Links auch via filter() und startsWith() beschaffen.
